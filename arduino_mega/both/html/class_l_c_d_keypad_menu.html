Detailed Description

═══════════════════════════════════════════════════════════════════════════════ LCD.H - Keypad & LCD Menu Interface (I²C @ 0x27, 20x4 characters) ═══════════════════════════════════════════════════════════════════════════════

LCDKeypadMenu provides a hardware menu interface using a 4×3 matrix keypad and 20×4 I²C LCD display. This is the primary user interface for standalone operation (no PC connection needed).
* Keypad Layout: 1 2 3 Keys 1-5 : Configure relay settings 4 5 6 Key 7 : Toggle menu ↔ protection mode 7 8 9 Key 9 : View status

  • 0 # Key 0 : Go back Key # : Reset latch or confirm entry Key * : Enter decimal point during numeric entry

Features:

  • Real-time LCD display updates
  • Numeric entry with decimal support
  • Input validation with error messages
  • Protection mode status display during operation

Usage:

  1. Create instance: LCDKeypadMenu menu(lcd_ref, shared_state);
    * 2. Call init() in setup()
    * 3. Call processInput(key) in loop() for each keypad press

Process a single keypad character This is the main state machine dispatcher routing input to appropriate handlers

Current menu state determines how character is interpreted:

  • In MENU_MAIN: 1-5 for settings, # for reset, 9 for status
  • In other menus: direct selection or numeric entry with # to confirm
