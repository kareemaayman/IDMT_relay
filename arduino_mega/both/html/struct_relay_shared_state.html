Detailed Description

═══════════════════════════════════════════════════════════════ RELAY_STATE.H - Shared State Structure for Menu Synchronization ═══════════════════════════════════════════════════════════════

This structure allows both the Serial menu (RelayMenu) and the LCD keypad menu (LCDKeypadMenu) to access and modify the same relay configuration parameters. By using pointers, changes made in one menu are immediately visible to the other.

Prevents duplicate state and ensures consistency across interfaces. Shared state holder for both menu systems

All members are pointers to global variables in both.ino. This allows both menus to access and modify the same values.

protection_enabled : Pointer to flag controlling whether protection is active latched : Pointer to latch flag (relay stuck open after trip)

std : Pointer to selected standard (IEC or IEEE) iec : Pointer to selected IEC curve (if STD_IEC is chosen) ieee : Pointer to selected IEEE curve (if STD_IEEE is chosen) tms : Pointer to TMS/TDS value (0.01-10.0) pickup : Pointer to pickup current in Amperes (> 0) inst_m : Pointer to instantaneous multiple (>= 1.01)
