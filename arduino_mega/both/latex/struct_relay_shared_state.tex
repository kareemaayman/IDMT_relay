\doxysubsection{Detailed Description}
═══════════════════════════════════════════════════════════════ \doxyref{RELAY\+\_\+\+STATE.\+H}{p.}{relay__state_8h_source} -\/ Shared State Structure for Menu Synchronization ═══════════════════════════════════════════════════════════════

This structure allows both the Serial menu (\doxyref{Relay\+Menu}{p.}{class_relay_menu}) and the LCD keypad menu (\doxyref{LCDKeypad\+Menu}{p.}{class_l_c_d_keypad_menu}) to access and modify the same relay configuration parameters. By using pointers, changes made in one menu are immediately visible to the other.

Prevents duplicate state and ensures consistency across interfaces. Shared state holder for both menu systems

All members are pointers to global variables in both.\+ino. This allows both menus to access and modify the same values.

protection\+\_\+enabled :\+ Pointer to flag controlling whether protection is active latched :\+ Pointer to latch flag (relay stuck open after trip)

std :\+ Pointer to selected standard (IEC or IEEE) iec :\+ Pointer to selected IEC curve (if STD\+\_\+\+IEC is chosen) ieee :\+ Pointer to selected IEEE curve (if STD\+\_\+\+IEEE is chosen) tms :\+ Pointer to TMS/\+\+TDS value (0.\+01-\/10.\+0) pickup :\+ Pointer to pickup current in Amperes ($>$ 0) inst\+\_\+m :\+ Pointer to instantaneous multiple ($>$= 1.\+01)
